# Paamalai

A cross-platform Bible app (iOS + Android) for English and Tamil readers.

**Two personas**

1. **Yearly reader** — a structured plan to finish the Bible in one year.
2. **Devotional reader** — a short daily devotion personalized to the reader's interests, generated by Claude.

Both personas can listen to chapters and devotions via on-device text-to-speech, in their preferred language.

## Stack

| | |
|---|---|
| Mobile | Flutter (Dart) — single codebase for iOS + Android |
| State / nav | `flutter_riverpod`, `go_router` |
| Local DB | `drift` over `sqlite3_flutter_libs` (read-only bundled Bible + writable progress mirror) |
| TTS | `flutter_tts` (device-native AVSpeechSynthesizer / Android TextToSpeech) |
| Backend | Supabase — Postgres, Auth, Edge Functions, Storage |
| AI | Anthropic Claude API (`claude-haiku-4-5-20251001`, prompt caching) — invoked from a Supabase Edge Function only |

## Required SDKs

Install once on a dev machine:

| Tool | Why | Install |
|---|---|---|
| Flutter ≥ 3.22 | Build the mobile app | <https://docs.flutter.dev/get-started/install> |
| Supabase CLI ≥ 1.180 | Run migrations + deploy edge functions locally | <https://supabase.com/docs/guides/local-development/cli/getting-started> |
| Deno ≥ 1.40 | Pulled in by the Supabase CLI for edge function runtime | <https://docs.deno.com/runtime/getting_started/installation/> |

After installing Flutter, run `flutter doctor` and resolve any issues for iOS (Xcode) and Android (Android Studio + an SDK).

## Day-to-day commands

> All commands assume you are at the repo root.

```bash
# Mobile app
cd app
flutter pub get
flutter run                       # picks any connected device / sim
flutter test                      # unit + widget tests
flutter analyze                   # static analysis

# Supabase (local)
supabase start                    # boots local Postgres + Studio at :54323
supabase db reset                 # re-runs migrations on local DB
supabase functions serve generate-devotion --env-file supabase/.env.local

# Bible data import (one-shot, regenerates assets/bible/*.sqlite)
cd tools
dart run build_bible_db.dart
```

### Environment

The app reads its Supabase config via `--dart-define` at build time:

```bash
flutter run \
  --dart-define=SUPABASE_URL=https://<project>.supabase.co \
  --dart-define=SUPABASE_ANON_KEY=<anon-key>
```

The Anthropic API key **never** ships with the app. It lives only as a Supabase function secret:

```bash
supabase secrets set ANTHROPIC_API_KEY=<key>
```
