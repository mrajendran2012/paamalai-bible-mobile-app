# 0003 — Daily Devotion (Claude-generated)

## Context

Persona P2 wants a short daily devotion that meets them where they are — biased to the topics they care about (anxiety, marriage, work, etc.) and anchored on a Bible passage. We generate this on-demand via Claude, cache it per user/day/language, and render it as Markdown.

## Personas

P2 (Devotional Reader). P3 (Listener) consumes the same content via TTS (see 0004).

## Functional requirements

### FR-DD-01 — Daily personalized devotion
- *As a* P2 user *I want* a short daily reflection tied to my interests *so that* my time in scripture connects to my life.
- **Given** interests are set and the user has internet, **When** they open the Devotion tab on a given calendar day, **Then** a ≤400-word devotion in their preferred language is shown, anchored on today's plan passage (or a curated default passage if no plan), and biased toward their interest tags.

### FR-DD-02 — Single generation per day
- **Given** a devotion has been generated for `(user_id, for_date, language)`, **When** the user reopens that day, **Then** the cached devotion is served and **no** new Claude API call is made.

### FR-DD-03 — Re-roll
- **Given** the user dislikes today's devotion, **When** they tap *Generate another*, **Then** a new devotion is generated and replaces the cached one.
- **Rate limit:** at most 3 re-rolls per user per calendar day. The 4th attempt returns `429 RATE_LIMITED`.

### FR-DD-04 — Offline graceful degrade
- **Given** the device is offline and no devotion is cached for today, **When** the user opens the Devotion tab, **Then** a friendly empty state ("Your devotion will be ready when you're back online") is shown.
- The last 7 days' previously-cached devotions remain readable.

### FR-DD-05 — AI disclosure
- **Given** any devotion is rendered, **Then** a small footer reads "Generated by Anthropic Claude" with a link to a brief explainer.

## Non-functional

- NFR-PERF-03: p95 end-to-end devotion generation ≤ 6 s.
- NFR-COST: ≤ ~US$0.005 per user-day at p50 (Haiku 4.5 + prompt caching on the system prompt).
- NFR-SEC: Anthropic API key lives only as the Supabase Edge Function secret `ANTHROPIC_API_KEY`. The mobile app never sees it.

## Data contracts

### Server-side: `POST /functions/v1/generate-devotion`

**Auth:** Supabase JWT (anonymous or authed user).

**Request**
```json
{ "for_date": "2026-04-25", "language": "en", "force_regenerate": false }
```

**Response 200**
```json
{
  "for_date": "2026-04-25",
  "language": "en",
  "passage_ref": "Romans 8:18-30",
  "body_md": "## Hope in the Waiting\n\n...",
  "model": "claude-haiku-4-5-20251001",
  "cached": true
}
```

**Errors**
| Code | Body | When |
|------|------|------|
| 401  | `{"error":"unauthorized"}` | missing/invalid JWT |
| 409  | `{"error":"NO_INTERESTS"}` | user has no interests set |
| 429  | `{"error":"RATE_LIMITED"}` | `force_regenerate=true` exceeded 3/day |
| 503  | `{"error":"UPSTREAM"}` | Anthropic call failed |

**Function behavior (deterministic order)**
1. Resolve `user_id` from JWT.
2. If `force_regenerate=false` and a row exists for `(user_id, for_date, language)`, return it with `cached: true`. **Do not call Claude.**
3. Read interests; if empty → return 409.
4. Resolve today's passage: today's yearly-plan first chapter if a plan exists; otherwise pick from a curated rotation (see §Curated fallback).
5. Call Anthropic Messages API with `claude-haiku-4-5-20251001`, **prompt caching enabled** on the system prompt (devotion-writing rubric + style guide).
6. Persist to `devotions_cache` (upsert if `force_regenerate=true`).
7. Return.

### Devotion shape (Markdown contract)

The `body_md` field is Markdown, ≤400 words, structured as:
- An H2 title (catchy but not clickbait).
- 2–4 short paragraphs of reflection that name the user's interest tags naturally (not as a list).
- One **Pray** section (single short paragraph).
- One **Reflect** section (1–3 short questions as a Markdown list).
- No images, tables, or links.

The system prompt enforces this structure and is the prompt-cache target (cost optimization).

### Curated fallback passages

Stored in `supabase/functions/generate-devotion/passages.ts` — ~120 `(passage_ref, theme)` entries rotated by `for_date` modulo length when the user has no active plan.

## Out of scope

- Saving / favoriting devotions (deferred)
- Sharing devotions externally
- Translation between EN and TA on the fly — devotions are generated in the requested language directly

## Risks / open questions

1. **Cost spike from re-rolls** — capped at 3/day per user. Monitor in week 1.
2. **Quality drift across languages** — Tamil-language devotions need a periodic spot-check by a native speaker; surface flagging in the UI later.
3. **AI safety / theological tone** — system prompt explicitly instructs avoiding partisan or denominationally divisive framing; reviewed by project owner before launch.
